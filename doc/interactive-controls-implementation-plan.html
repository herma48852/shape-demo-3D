Implementation Plan: Native C++ 3D Shapes Interactive Controls

This document outlines the step-by-step engineering plan to replicate the HTML/JS interactive publisher and subscriber controls (from main11.html) inside the native C++ RTI Connext DDS environment provided in the archive.

To replace the HTML DOM UI, we will integrate Dear ImGui, an immediate-mode GUI library perfectly suited for real-time OpenGL/GLFW applications.

1 UI Framework Integration (Dear ImGui)

Your native app currently uses raw OpenGL and GLFW (src/graphics/graphics_engine.cpp). We need to overlay a UI panel on top of the 3D viewport.

  1. Update CMakeLists.txt:
    • Add FetchContent or a submodule to pull Dear ImGui into the build system.
    • Link the ImGui targets (imgui, imgui_impl_glfw, imgui_impl_opengl3) to your rti_3d_shapes executable.
  2. Update graphics_engine.hpp:
    • Add initialization and shutdown methods for ImGui.
    • Add render_ui() context hooks.
  3. Update graphics_engine.cpp:
    • In initialize(), add ImGui::CreateContext(), ImGui_ImplGlfw_InitForOpenGL(), and ImGui_ImplOpenGL3_Init().
    • In begin_frame(), add the ImGui new frame initializers.
    • In end_frame(), call ImGui::Render() and ImGui_ImplOpenGL3_RenderDrawData() just before glfwSwapBuffers().

2 Dynamic State Management (Replacing JS Arrays)

In main11.html, dynamic instances are stored in activePublishers and activeSubscribers arrays. We must replace the hardcoded kinematics in src/main.cpp with C++ equivalents.

  1. Define C++ Enums for Trajectories: 
    enum class TrajectoryType { PLANE_Z125, BOUNCE_3D, ORBIT };
  2. Define Publisher Struct (src/main.cpp):

    Create a struct to hold the physics state and DDS identifiers, mimicking the JS payload:

    struct LocalPublisher {
    std::string id;
    std::string shape;
    std::string color;
    int size;
    int publish_hz;
    TrajectoryType trajectory;

    // Kinematics
    float x, y, z;
    float vx, vy, vz;
    std::chrono::steady_clock::time_point last_publish_time;
};
  3. Define Subscriber Struct: 
    struct LocalSubscriber {
    std::string id;
    std::string target_shape;
    std::string color_filter;
    int time_filter_ms;
};
  4. Create State Vectors:

    Initialize std::vector<LocalPublisher> active_publishers; and std::vector<LocalSubscriber> active_subscribers; inside main().

4 Porting the Physics & Kinematics Engine

The physics logic currently hardcoded in main.cpp for one square and one circle needs to be generalized to loop over the active_publishers vector, perfectly matching the physics logic in main11.html.

  1. Time Delta Calculation:

    Calculate dt and global_time_counter at the top of the main loop to drive smooth physics regardless of frame rate.

  2. The Kinematics Loop: 
    for (auto& pub : active_publishers) {
    if (pub.trajectory == TrajectoryType::PLANE_Z125) {
        pub.x += pub.vx; pub.y += pub.vy;
        // Apply 2D wall bounds check
    } 
    else if (pub.trajectory == TrajectoryType::BOUNCE_3D) {
        pub.x += pub.vx; pub.y += pub.vy; pub.z += pub.vz;
        // Apply 3D wall bounds check (0 to 250 on all axes)
    }
    else if (pub.trajectory == TrajectoryType::ORBIT) {
        // Apply deterministic trigonometric path based on global_time_counter
        pub.x = 125.0f + std::cos(global_time) * 80.0f;
        // ... apply Y and Z logic from README.html
    }

    // Publish logic based on pub.publish_hz
    if (time_since_last_publish >= (1.0f / pub.publish_hz)) {
        dds_manager.write_shape_sample(pub.shape, pub.color, pub.x, pub.y, pub.z, pub.size, ...);
    }
}

5 Rendering the Dynamic Data

Finally, the GraphicsEngine needs to render the dynamic data instead of hardcoded primitive calls.

  1. Render Local Writers (Solid Shapes):

    Iterate through active_publishers and call graphics.draw_cube, draw_sphere, etc., at their current x, y, z coordinates.

  2. Render Remote Readers (Wireframes & Trails):
    • Iterate through the shape_registry (data populated by sample_queue.drain_queue()).
    • Check if the incoming shape matches any filter in active_subscribers.
    • If it matches, render the wireframe representation (graphics.draw_cube(..., wireframe=true)).
    • To implement the "Receive Cache Size" trail (Ghost meshes), maintain a circular buffer of the last N coordinates for each discovered instance in shape_registry and render them with decreasing opacity.
